# Architecture diagram — provenance

The architecture diagram is **generated**, not hand-drawn:

```text
broker.netjson  ──(netjson-diagrams CLI)──▶  broker.puml  ──(PlantUML, offline)──▶  broker.svg
```

- **`broker.netjson`** — source of truth. A NetJSON `NetworkGraph` describing the broker topology
  (clients, Queue API, Processor, SQL Server) and the flows between them. Edit this, never the outputs.
- **`broker.puml`** — generated by the sibling [`netjson-diagrams`](https://ariugwu.com/netjson) CLI.
- **`broker.svg`** — rendered by [PlantUML](https://plantuml.com) (MIT build) using the **Smetana**
  layout engine, so no Graphviz is required and nothing is fetched at render time.

Regenerate both outputs with:

```bash
./docs/scripts/render-diagram.sh
```

## Trust / offline notes

- The PlantUML jar is the **MIT-licensed** build, pinned by version **and** SHA-256 in
  `render-diagram.sh`, downloaded once into `docs/.cache/` (gitignored).
- Sprite `!include` directives resolve from the jar's bundled stdlib — there are **no calls** to
  `plantuml.com` or `kroki.io` at render time, and the committed `broker.svg` is a static asset.
- This mirrors the offline / no-telemetry posture of `netjson-diagrams` itself.
