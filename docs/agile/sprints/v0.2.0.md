### Post-mortem

Written at sprint close, 2026-06-08. Each agent that owned a card contributes a bullet.

**What went well?**

- (dotnet-architect) The `ILogShipper` abstraction was designed cleanly in a single pass: the
  `BrokerLogEvent` record captured every dimension needed (Timestamp, EventType, Level, Source,
  CorrelationId, ClientId, RequestType, Message, Properties, Exception) without overfit. The
  `CompositeLogShipper` fan-out pattern meant sink combinations required no additional code.
- (dotnet-architect) Config-driven sink selection (`LogShipper:Sink` in appsettings) kept all
  service wiring in one place and made the Seq profile additive — no changes to Queue or Processor
  startup when switching sinks.
- (docs-web) The docs article and the implementation were authored in the same pass. The six event
  types, sink config table, and custom-shipper extension point were straightforward to document
  because the abstraction was clean. Wiring into `toc.yml` was a one-liner.
- (dotnet-architect + docs-web) Parallel agent execution for service wiring and docs reduced
  wall-clock time for the L card significantly — effectively converted it to 2M in practice.

**What could improve?**

- (test-lead) The test harness broke when the `SubmitRequestConsumer` constructor was extended to
  accept `ILogShipper`. The fix (register `NullLogShipper`) was straightforward, but it was caught
  only at build/test time rather than up front. Going forward: when DI constructors change in
  consumers or handlers, the test harness update should be part of the same task, not a follow-up
  discovery.
- (dotnet-architect) `ElsaBroker.WorkflowServer` was intentionally skipped for `ILogShipper`
  wiring — it is a thin custom Elsa host and its log volume is low. This is a documented gap, not
  an oversight, but the backlog should reflect it so the next team is aware.

**Process changes adopted**

- None needed this sprint. The v0.1.0 process changes (post-mortem step, close → post-mortem →
  propose-next as a single sequence) are working well and required no adjustment.
